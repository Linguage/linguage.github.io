Tactics

Claude Code 深度实操手册

02

01. 谋定后动

不要直接开始对话，这是新手的通病。强制自己先进入 Plan Mode。在那里的五分钟深思熟虑，能省下后续数小时的调试地狱。架构设计不仅是给结果，更是为模型划定清晰的执行路径。

02. 入职文档

将 CLAUDE.md 视为新员工的入职手册。不要写成长篇小说，严格限制在 200 条指令内。告诉模型“为什么”这样做（Context），它会比单纯执行命令更聪明地处理边缘情况。

03. 记忆止损

模型在上下文占用达到 30% 时就开始显现疲态。坚持单一会话只做单一功能。当逻辑开始混乱时，不要恋战，复制关键信息，使用 /clear 彻底重置，始终保持上下文的纯净度。

04. 负向约束

Claude 4.5 喜欢过度设计。必须明确告诉它“不要做什么”。明确要求：保持简单，不要增加未请求的抽象层，不要为了灵活性而牺牲可读性。用约束条件构筑围栏。

05. 双模切换

利用 Opus 4.5 的深度推理能力进行架构规划和权衡，随后切换至 Sonnet 3.5 (Shift+Tab) 进行具体的代码实现。让昂贵的模型做大脑，让快速的模型做手脚，这是最优解。

06. 自动化闭环

Claude Code 不仅是聊天机器人，更是系统组件。利用 Headless 模式 (-p) 串联 Bash 脚本。建立反馈闭环：错误触发日志，日志修正文档，文档优化下一次输出。
