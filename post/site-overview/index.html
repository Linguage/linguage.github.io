Linguista 是一个什么样的网站

amp-smart · 2026-03-14

不是博客，是知识门户

Linguista（linguista.cn）从外表看是一个 Hugo 静态站点，但它和大多数个人博客不太一样。它不是按时间倒序排列文章的日记本，而是一个围绕几个核心主题持续生长的内容系统。这些主题包括 AI 与智能体工作流、技术工具与软件工程、数学与科学方法、优秀人物的文章和思想，以及高质量课程资料的中文整理。

这个站点的起点是个人阅读。我是一个兴趣广泛的阅读者，涉猎科技、历史、哲学、文学等领域，最近几年对大语言模型和基于 AI 的开发方式尤其着迷。阅读过程中积累了大量值得反复回看的内容，于是自然地想把它们系统化地整理出来、分享出去。Linguista 就是这个想法的产物。

站点的首页不展示最新文章，而是扮演导航门户的角色，把读者引导到不同的内容入口。这个设计选择反映了一个基本判断：对于一个内容类型丰富的站点，“最新发布"不一定是读者最需要看到的东西，“找到自己感兴趣的板块"才是。

六个内容板块

站点目前有六个主要板块，各自承担不同的功能。

info（信息库） 是更新最频繁、内容量最大的板块。它包含每日文摘（digest）、TL;DR 摘要卡片（tldrcards）和信息图卡片文章（htmlcards）。这个板块的核心功能是快速吸收优质信息，并把它们沉淀为可检索的知识卡片。如果你想在短时间内获取经过筛选的高质量内容，这是最合适的入口。

person（人物选集） 是站点辨识度最高的板块之一。目前主要收录了 Paul Graham、Naval Ravikant 和陶哲轩三位作者的文章与思想。每位作者都有比较完整的内部结构，比如 Paul Graham 的部分同时提供英文原文和中文译文，陶哲轩的部分则按"职业建议"和"写作指南"两个主题组织。这个板块体现的思路是：选定少数真正值得深读的人物，然后系统化地整理他们的作品。

courses（课程资料） 用于整理优质课程内容，目前偏重 AI 和 Agent 相关主题。已经整理的课程包括 Andrew Ng 的 Agentic AI、Gemini CLI、Build with Andrew 以及 Anthropic 的 Agent Skills 课程。其中 Andrew Ng 的课程结构最完整，拆分了概览、专题博客、学习指南、视频实录和课程脚本等多个维度。这个板块的定位更接近"学习资料站”，而不是简单的听课笔记。

labs（实验工坊） 是最偏原创实践的板块，分为 workshop（工程实践与项目复盘）和 notes-dr（专题研究笔记）两部分。如果说 courses 偏向学习输入，labs 就偏向实践输出——记录自己的实验过程、项目开发经历和方法探索。

rosetta（知识摘编） 是一个跨学科的内容区，收录访谈、演讲、技术文章与人文主题。它下设 humanity、technology 和 chat-notes 三个子区，其中 chat-notes 体量最大，更像是一个高质量长对话的整理库。这个板块的位置介于 info 和 person 之间：比 info 更深更长，比 person 更杂更广。

post（站点更新） 是你现在正在阅读的这个板块。它承担站点内容更新说明、技术架构调整记录和建站过程复盘等功能。随着站点的持续演进，这个板块会逐步成为连接读者与站点发展过程的重要窗口。

四种阅读路径

从读者使用场景来看，站点大致提供四种进入方式：

想快速获取高质量信息的读者，适合从 info 开始。想系统学习某个主题的读者，适合从 courses 进入。想做深度阅读、接触优秀思想者作品的读者，适合去 person 和 rosetta。想了解站点本身的技术实现和演进过程，可以来 labs 和 post。

这四条路径并不互斥，很多读者会在不同时候走不同的路线。但它们反映了站点内容组织的基本逻辑：按读者意图分流，而不是把所有内容混在一条时间线里。

内容策展而非随意发布

这个站点有一个贯穿始终的特点：它不是随手记录的地方，而是经过筛选和整理的内容集合。

我对自己发布的内容有一个基本要求——至少是自己愿意多读几遍的东西。无论是原创文章、翻译作品还是搬运分享的内容，都会经过一轮筛选和加工。大量内容从原始素材开始，经过中译、摘要、主题整理和专题归类，最终变成结构化的页面。

这种做法的代价是更新速度不会太快，但好处是每一篇内容都有相对稳定的长期价值。站点追求的是"可检索的知识沉淀”，而不是"持续的信息流"。

前台体验上的一些设计

在呈现方式上，站点也做了一些超出普通静态博客的设计。首页承担门户导航功能，多个板块使用 tab 结构组织子栏目，列表页普遍采用卡片化展示，部分内容区使用了热力图和分页机制，还支持搜索、目录树和可折叠导航等功能。

这些设计的出发点是同一个：当内容类型足够丰富、数量足够多的时候，简单的文章列表已经不够用了，需要更接近"内容产品"的交互方式来帮助读者找到自己想看的东西。

一个持续生长的项目

Linguista 不是一个写完就放在那里的站点。它的内容在持续更新，结构也在随着内容的增长而调整。从最初的个人博客雏形，到现在的多板块知识门户，这个过程本身也是一种学习和实践。

如果用一句话来概括：这是一个以 AI、技术、学习与人物思想为核心主题的中文内容站点，兼具信息精选、专题文集、课程资料、实验记录和知识归档功能。它更接近一个持续生长的个人知识门户，而不是传统意义上的博客。

欢迎探索，也欢迎通过评论区或社交媒体与我交流。
